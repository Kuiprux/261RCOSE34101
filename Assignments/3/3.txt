1-5: 코드에 사용되는 함수들이 정의된 헤더 파일을 컴파일 시 포함하게끔 한다

9-12: 코드에 사용되는 변수를 정의한다
16: fork()로 동일한 코드를 사용하는 자식 프로세스를 생성한다
19: 부모 프로세스와 자식 프로세스, 그리고 fork의 오류 여부를 구분하기 위해 fork의 리턴 값을 switch문으로 경우를 나눠 처리하게끔 한다
20-22: fork가 실패한 경우 -1이 리턴되므로, 그 경우는 실패함을 알리고 프로그램을 종료한다.
23: fork에서 리턴받은 값이 -1이나 0이 아닌 값인 경우가 모두 여기에 해당하는데, 이는 값이 양수인 경우로, 부모 함수에서 자식 함수를 성공적으로 생성한 경우 이 case 구문이 실행된다.
25: fork에서 리턴받은 pid는 자식의 pid이므로, getpid()로 자신의 pid를 얻어온다
26: 부모 프로세스(자신)의 pid를 출력한다..
29: 자식 프로세스의 favorite_number 값을 받아와야 하므로, 자식 프로세스가 해당 값을 리턴하며 종료할 때까지 대기한다.
31: 자식 프로세스가 종료될 때 넘겨준 값이 status에 저장되었으므로, 그 값을 출력한다. 자식 프로세스가 정상 종료된 경우 자식이 넘겨준 값은 8-15비트에 저장되며 0-7비트는 0이므로 256으로 나누어 원래의 값을 얻어낸다.
33: fork의 리턴값이 0인 경우로, 부모 프로세스의 fork()로 생성된 자식 프로세스가 동일한 구문을 실행하면 추가로 자식 프로세스가 생성되지 않고 0이 리턴되므로 자식 프로세스가 이 case 구문을 실행하게 된다.
35: 다중 프로세스의 코드 실행 순서는 보장할 수 없어 자식 프로세스의 출력 구문이 부모 프로세스의 출력 구문보다 먼저 실행될 가능성을 배제하기 위해 앞서 정의한 SLEEP_TIME만큼 프로세스 실행을 잠시 중단한다.
37: favorite_number의 값을 지정한다. 이 코드는 자식 프로세스에서만 실행되기 때문에, 부모 프로세스에서는 이 변경된 값을 직접 알아낼 수 없고, 자식 프로세스가 종료될 때 이 값을 부모 프로세스에게 넘겨주어야 한다.
40, 43: 각각 자식 프로세스(자신)과 부모 프로세스의 pid를 알아낸다.
45: 자식 프로세스(자신)과 부모 프로세스의 pid를 출력한다.
46: favorite_fruit와 favorite_number를 출력한다. 여기서 favorite_fruit는 switch 구문 이전에 값이 할당되었으므로 자식 프로세스에서도 바로 접근할 수 있다.
49: 자식 프로세스(자신)의 실행을 종료하며 favorite_number 값을 전달한다. 그러고 나면 부모 프로세스의 29번 구문의 blocking이 해제되며 favorite_number*256이 status에 저장될 것이다.
